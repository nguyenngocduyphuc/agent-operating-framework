Agent Operating Framework (AOF)

Lớp tin cậy cho AI agent — chặn sai chỗ, bắt bằng chứng, bàn giao, đo hiệu quả. Không phải orchestrator · v0.3.0b1 · MIT
TEAM ONE-PAGER

1. Vì sao tồn tại

  • Agent hay sai repo/nhánh, tự bảo xong, giẫm task, mất bàn giao.
  • AOF đứng trước mọi agent (Claude / Cowork / Codex…): fail-closed.
  • CEO no-code: 1 câu giao việc → đôi khi duyệt → đọc recap / hiệu quả.

2. Giá trị (1 dòng / người)

  • CEO: “Tình hình?” → 4 trạng thái + bước tiếp + pulse 24h.
  • Agent: preflight · contract+Karpathy · lease · verify · scope.
  • Team: CI + Issue hiệu quả + release tag.
  • Đã đo: chặn out-of-scope 100% vs bare 43% (n=105); cost ~+35%.

3. Workflow no-code (9 bước máy)

preflight → check_contract (Karpathy) → [needs_approval?] → work → verify_gate + audit_scope → post_evidence (Done|Blocked) → session_handoff + RECAP → (phiên sau) aof_resume → status_report / estate_report · auto HIEU_QUA_HOM_NAY.md
