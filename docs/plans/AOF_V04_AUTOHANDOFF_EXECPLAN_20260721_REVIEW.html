5 tính năng — acceptance criteria + file đụng tới

F2 — Handoff tự kèm recapnhỏ nhất checkpoint 1

Sửa nhánh session_handoff trong mcp_server.py (và lệnh aof handoff trong cli.py) để mỗi lần gọi sinh cả recap HTML cùng stamp, link chéo ở đầu file md.

Acceptance: gọi session_handoff một lần → 2 file HANDOFF_<stamp>.md + RECAP_<stamp>.html cùng stamp, md có dòng Recap: ./RECAP_<stamp>.html.

File: core/mcp_server.py, core/cli.py, core/oplog.py, tests/test_recap_handoff.py

F1 — Đa-repo + index + aof resume lõi wavecheckpoint 2

Ghi handoff/recap vào repo_identity(bound_cwd) thay vì AOF_WORKSPACE cố định; append dòng vào ~/.aof/handoffs/index.jsonl mỗi lần ghi; tool MCP thứ 13 + lệnh CLI aof resume đọc index → in RESUME BRIEF.

Acceptance: làm trong repo A → file nằm trong A/docs/sessions; index có dòng mới đúng schema; aof resume --task X trả đúng brief; test đa-repo (2 repo giả lập, không tráo lẫn).

File: core/lease.py (dùng lại), core/oplog.py (index writer), core/mcp_server.py (tool 13 + sửa dòng 787), core/cli.py (subparser resume), tests mới test_resume_multirepo.py

F3 — Error ledger chống tái phạm

~/.aof/errors.jsonl qua session_log event error (không thêm tool mới); preflight WARN khi có lỗi status != closed hoặc fingerprint lặp ≥2; đóng lỗi bắt buộc test_ref; lệnh aof lessons.

Acceptance: seed 2 lỗi → preflight warn đúng; đóng lỗi thiếu test_ref bị từ chối; aof lessons liệt kê đúng lỗi mở + lỗi thiếu test_ref.

File: core/preflight.py, core/oplog.py hoặc core/errors.py (mới), core/cli.py (subparser lessons), tests mới test_error_ledger.py

F4 — Chuẩn kiểm soát worker

4 luật đo được thành policy key worker_stale_after_s (mặc định 300, theo khuôn DEFAULT_STALE_AFTER_S) + docs/WORKER_CONTROL_VI.md 1 trang; op_log --task đã có sẵn, chỉ cần xác nhận hiển thị đủ verify fail rate.

Acceptance: policy key mới có test; doc 1 trang tồn tại; op_log --task hiển thị đủ số liệu.

File: core/mcp_server.py (policy default), docs/WORKER_CONTROL_VI.md (mới), tests/test_policy_compat.py (mở rộng)

F5 — CIchặn hồi quy

.github/workflows/ci.yml: ruff + pytest, ma trận Python 3.10/3.11/3.12, chạy trên push/PR. Không đụng release/publish.

Acceptance: file YAML hợp lệ, chạy được cục bộ qua act nếu có, hoặc ít nhất lint YAML sạch.

File: .github/workflows/ci.yml (mới)
